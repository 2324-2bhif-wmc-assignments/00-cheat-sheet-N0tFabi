Web Server

What is a web server?


A Web Server is a Server that can respond to multiple types of requests, mainly HTTP and HTTPS and return documents based on the properties of the request. A Server of this type is a Computer with a special Software. The Servers are stored either local or in a company.

What does a web server?


A Web Server listens for request to (mainly) read the content of Documents that are stored on the Web Server. However, to prevent some suspicious person to access data they shouldn't have access to, the Web Server also checks if the request is valid, so is the Headers / Body content of the request valid? Based on whether the Server send / can send a result back to the user it returns a status code alongside the response which contains the requested document when the request is properly forged

How can content be stored on a web server?


There are several ways to store Documents on a Web Server, such as MySQL, file system or web storage just to name a few. So one can use a Database, recommended for dynamic data, file system to store documents outside of the web root. This is used for storing files like images, stylesheets or JavaScript files.

How is content in webpages described?


The static content of a Website is saved in .html (HyperText Markup Language) files. Dynamic content, for example the language of a website controlled by user input, can be written in PHP (which in the end also generates a .html file).

Web Client

What is a Web Client?

The Web Client is the Front-End and allows the user to interact with the Web Server. A Web Client is an application, such as Firefox or Edge, that sends these Requests to the Web Servers. Web Clients send the request to the nearest Web Server, where the requested data is most likely already stored from Cloudflare, and then, after validating the request, returning the requested data.

What does a web client?

The Web Client sends the Request to the Web Server to get the Documents the User wants to access via their URL. The Web Client is also responsible for displaying the html, css and js on the website, handling errors, for example if a domain is not reachable or does not exist. But how does the Web client know what IP he has to ping to get the website's content? For that, the Web Client first sends a DNS Server which turns the URL into an IP-Address the Web Client actually can work with.

Web Client and Web Browser is the same

The Web Client and Web Browser are two different things. The Web Browser is the "visualisation" of the Web Client. The Web Browser inherits the Web Client and visualizes the results the Web Client brings. The Web Client is only responsible for receiving the data from the Web Server.
